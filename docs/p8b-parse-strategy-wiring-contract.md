# P8B 工作空间解析策略接线契约

## 1. 目标与真实基线

P8B 把已有工作空间设置 `parseStrategy=light|local|ask` 接到技术标和商务标的实际解析入口，使用户选择的默认策略真正决定本次解析动作。

当前真实基线是：设置已持久化 `parseStrategy`，但技术标始终手动创建轻量解析任务，商务标上传后也始终自动创建轻量解析任务；服务端生产注册表仅有 `lightweight`。本包只补齐策略决策与本地回传入口，不改变既有解析引擎调度契约。

## 2. 三种策略的固定语义

| 工作空间策略 | 技术标“解析”动作 | 商务标上传后的动作与“整段重解析” | 是否创建 `parse` 任务 |
|---|---|---|---:|
| `light` | 以既有 `payload.engine=lightweight` 创建任务 | 上传后、重解析时均以同一方式创建轻量任务 | 是 |
| `local` | 不创建任务，进入本地解析回传页 | 上传完成后或重解析时进入本地解析回传页 | 否 |
| `ask` | 不创建任务，先显示一次性方式选择框 | 上传完成后或重解析时先显示同一选择框 | 由用户本次选择决定 |

`ask` 中的选择只作用于当前一次解析，不回写或修改工作空间默认设置。选择“在线轻量解析”与 `light` 完全一致；选择“本地 MinerU 回传”与 `local` 完全一致。关闭选择框、策略读取失败、项目 ID 为空或当前流水线繁忙时，均不得创建解析任务。

本地路径固定为 `/local-parser?projectId=<当前项目 ID>`。本地解析页必须读取该查询参数并预填项目 ID，用户仍须自行在保密机运行 MinerU、粘贴 Markdown 并调用既有 `POST /api/projects/{id}/parse-callback`。跳转本身不上传原文件、不调用回调、不写编辑态。

## 3. 策略读取接口与权限

新增唯一策略读取接口：

| 方法 | 路径 | 成功响应 | 缓存 |
|---|---|---|---|
| GET | `/api/settings/parse-strategy` | `{ "parseStrategy": "light" }` | `Cache-Control: no-store` |

它只能返回上述单一字段，值仅为 `light`、`local` 或 `ask`。不得返回 `apiKey`、`apiBaseUrl`、provider、model、embedding、导出模板、更新时间、工作空间 ID 或其他设置字段。

认证模式为 `required` 时，该接口使用既有标书制作者工作空间解析规则：精确 `bid_writer` 可读当前成员工作空间；未登录为 `401 auth_required`；财务、人力、投标人和非所有者的其他非标书制作者角色为 `403 role_forbidden`；非成员 `X-Workspace-Id` 为 `403 workspace_forbidden`。认证模式为 `disabled` 保持既有个人版工作空间选择语义。`GET /api/settings` 与 `PUT /api/settings` 的所有者限制不变。

接口只读取已保存策略；若该空间尚无设置行，返回既有默认值 `light`，且不得因读取而新建设置行或修改数据库。客户端只在 React 内存中保存本次读取结果，点击解析时必须重新读取，不能使用 `localStorage`、`sessionStorage` 或旧设置缓存决定策略。

## 4. 前端边界、状态与错误处理

技术标和商务标都复用同一策略读取和一次性选择组件；不得复制两套策略映射，也不得修改 `useProjectPipeline` 的任务协议。轻量路径必须显式传递 `engine: "lightweight"`，以保留 P8 的结果可追溯性；不得向任务传入 `local`、`ask`、`mineru`、`docling` 或任意用户输入的引擎名。

策略读取中，解析入口显示中文“正在读取解析策略”；读取失败显示固定中文“暂时无法读取解析策略，请稍后重试”，且不能回显后端详情、URL、配置或 Key。`local` 与 `ask` 的本地选项必须明确说明“本地回传，不在服务器启动 MinerU”。

商务标的自动解析行为仅改为“上传后按当前策略处理”：`light` 仍自动解析；`local` 和 `ask` 只在文件成功上传后跳转或显示选择，不创建任务。技术标仍由用户点击解析入口发起。两种工作台的后续编辑态刷新、成功提示、任务失败提示和项目步骤推进均复用既有轻量任务路径。

## 5. 安全与明确非目标

本包不得：

- 安装、内嵌、启动或通过 `subprocess`、Shell、网络请求调用 MinerU、Docling 或其他外部解析器；
- 修改 `parse_engines` 生产注册表、`task_service._run_parse`、`parse-callback` Token 默认策略、解析文件格式、数据库模型、迁移或依赖；
- 创建本地解析任务、把 `local`/`ask` 映射为伪造 engine，或对策略读取失败静默降级为轻量解析；
- 放宽设置读写权限、暴露完整设置、改动财务、人力、投标人权限或认证中间件；
- 自动提交本地回传内容、自动调用分析步骤、修改 `parseStrategy`，或把策略写入浏览器持久化存储。

所有新建或主要修改的公开模块、组件、Hook、函数和测试都必须含中文“模块 / 用途 / 对接 / 二次开发”四字段注释。

## 6. 验收矩阵

后端测试必须覆盖默认无设置行只返回 `light` 且不建行、三种已保存策略、精确字段白名单与 `no-store`、required 下标书制作者/未登录/非标书制作者/跨空间，以及 disabled 兼容路径。

前端 Playwright E2E 必须在真实本机前后端中覆盖：

1. 技术标 `light` 创建 `engine=lightweight` 解析任务并写入解析结果；
2. 技术标 `local` 只跳到带项目 ID 的本地回传页，整个页面流程没有 `POST .../tasks`；
3. 技术标 `ask` 先显示选择框，关闭不创建任务；选择轻量任务与选择本地回传均符合契约；
4. 商务标上传和“整段重解析”同样遵循 `local`/`ask`，不再无条件自动轻量解析；
5. 策略接口失败不创建任务、不泄漏错误详情；浏览器不写策略持久化数据；请求只包含认证、策略读取、项目文件/任务/编辑态等实现本场景必需接口，且没有外部请求。

通过定向后端测试、前端 lint/build、P8B E2E、原有解析相关后端回归、原有商务标与技术标相关 E2E及 `git diff --check` 后，才能进行文档闭环和协作分支推送。
