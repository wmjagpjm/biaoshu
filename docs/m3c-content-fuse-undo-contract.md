# M3-C 融合写入单批撤销契约

> **交付状态**：已按本契约完成并推送。计划=`c63310f`，实现=`b8ff605`；M3-B/M3-C E2E 6 passed、M3-A E2E 1 passed、P10H 回归 10 passed、单 worker 串行全量 106 passed，lint/build 通过。

## 1. 审计结论与方案选择

M3-B 已支持融合建议差异预览、逐项确认写入和编辑器自动保存，但明确遗留“写入后专用回滚/历史”。现有章节状态已在前端编辑器内存中管理，并经串行防抖 PUT 保存；因此首个安全闭环不需要新表、新 API 或持久化历史。

本包比较三种方案：

1. **当前对话框内、最近一批、一次性撤销（采用）**：状态最小，天然随项目和对话框销毁，能用当前章节快照阻止覆盖后续手工编辑。
2. 工作区级临时撤销：关闭对话框后仍保留，但需要把融合专属状态提升到页面，增加项目切换、重新加载和来源任务代次边界。
3. 后端版本历史：可跨刷新恢复，但需要新数据模型、保留策略、并发契约、权限与审计，不符合当前最小能力。

M3-C 采用方案 1，只解决“刚确认写入后发现不合适”的即时恢复。它不是通用撤销，也不声称提供历史版本。

## 2. 固定数据与生命周期

每次“确认写入所选”至少成功写入一条建议时，在 `ContentFuseDialog` 当前实例内保存一份**最近批次快照**。按目标章节 ID 去重，每章只含：

- `chapterId`；
- 写入时的精确标题；
- 写入前正文与状态；
- 本批最终写入后的正文与状态；
- 本批作用于该章的建议 ID 集合。

不得保存模板正文、卡片正文、提示词、模型响应、来源 ID、项目 ID、工作空间、用户、Cookie、CSRF 或其他章节字段。快照只存在 React 内存，不得写入 `localStorage`、`sessionStorage`、IndexedDB、URL、日志或后端。

生命周期固定为：打开对话框或切换项目时为空；生成新一轮建议时清空；一次成功确认写入后建立；后续又有成功确认写入时用最新批次替换；点击撤销后无论全部、部分或零项恢复，均立即消费；关闭对话框后销毁，重新打开不可恢复。

## 3. 写入与撤销规则

确认写入仍沿用 M3-B：点击瞬间校验建议 base，逐条调用编辑器替换函数，部分成功允许，未确认项、空建议、基线漂移和已写入建议均不写。

“撤销本次写入”只在存在未消费快照时显示。点击瞬间逐章重新读取当前章节，并同时满足以下条件才恢复：

1. 章节仍存在；
2. 当前标题与写入时标题完全一致；
3. 当前正文与本批最终写入正文完全一致；
4. 当前状态与本批最终写入状态完全一致。

满足时恢复写入前正文和写入前状态，并由既有编辑器重新派生 `preview`、`wordCount`，继续走同一串行防抖保存。任一条件不满足即视为后续编辑或结构漂移：该章跳过，绝不覆盖当前内容。允许部分恢复；消息固定显示“已撤销 N 章，跳过 M 章”。

仅成功恢复章节对应的建议可从“已写入”集合移除，从而在 base 重新匹配时允许再次勾选；被跳过章节仍保持已写入标记。撤销按钮只可执行一次，不做自动重试、二次确认或撤销的撤销。

## 4. 权限、网络与非目标

M3-C 不新增路由、角色或鉴权逻辑，仍只存在于既有标书制作者技术标工作区。除既有模板/卡片读取、`content_fuse` 任务和编辑器状态 PUT 外，不新增网络请求；撤销动作本身不得调用新端点。

本包不修改后端、数据库、任务协议、融合模型输出、响应矩阵、大纲、分析、事实、图片、模板、卡片或其他页面；不提供跨关闭、跨刷新、跨项目、跨设备、跨用户历史；不回滚手工编辑、AI 生成、修订预览或非本批融合变更；不新增浏览器持久化、导出、审计查看或外网。

## 5. 验收底线

E2E 至少覆盖：多章确认写入后一次撤销，正文与原状态恢复并经刷新保持；写入后手工修改一章时，另一章可恢复而漂移章保持手工内容；关闭再打开无撤销入口；撤销快照消费后按钮消失；撤销不产生新业务 API，仍只经既有编辑器状态 PUT 保存。回归必须覆盖原 M3-B 默认不勾选、base 漂移阻断、未确认关闭不写和删除目标章路径。
