# P9D 导出图片失效引用浏览器提示契约

> **交付状态**：已按本契约完成并推送。计划=`4925a51`，实现=`e5adad7`；后端项目图片专项 14 passed、P9D E2E 4 passed、前端单 worker 全量 110 passed，lint/build 通过。
> **工作分支**：`collab/grok-code-codex-review`。

## 1. 审计结论与方案选择

项目图片导出后端已经完成安全闭环：技术标章节正文和商务标整包 Markdown 都只解析独占行 `biaoshu-image://file_*`；无效语法、非法标识、不存在、跨项目、非图片角色或损坏文件均在 Word 中写入“图片引用无效”可见段落，并把同一原因数组放入成功任务的 `result.imageWarnings`。现有后端测试已经覆盖合法图片嵌入和无效引用降级。

当前缺口只在浏览器层：技术标和商务标导出成功后立即下载 Word，却完全忽略 `imageWarnings`，用户只有打开文档才能知道哪些配图未写入。

本包采用“导出页内、后端权威、非阻断告警”方案：两个页面复用一个纯展示组件和一个最小归一化函数；任务成功后先保存告警，再继续既有下载。它不重做图片解析，也不从告警文本推导文件、路径或链接。

## 2. 固定输入与显示规则

唯一输入是既有成功导出任务的 `result.imageWarnings`。前端把任务结果视为不可信结构，只接受数组中的字符串，并执行以下固定收敛：

- 去除首尾空白，丢弃空字符串和非字符串；
- 最多保留前 20 条；
- 每条最多显示 240 个 Unicode 字符；
- 不解析 Markdown、HTML、URL、文件 ID 或本机路径；
- 通过 React 文本节点渲染，不使用 `dangerouslySetInnerHTML`。

有告警时，当前导出页显示明确标题、条数和逐条原因，并说明“Word 已生成并继续下载，请在文档中检查降级位置”。无告警时不显示告警区。告警不改变成功状态、下载地址或文件内容。

每次开始新导出前必须清空上一次告警；项目 ID 变化时也必须清空，禁止把旧项目告警短暂显示到新项目。失败、取消或结果结构非法时不得伪造告警。

## 3. 技术标与商务标一致性

技术标继续通过 `pipeline.downloadExport(task)` 下载，商务标继续使用既有后端 `downloadPath`。两者必须在成功任务中应用相同归一化和相同告警组件；不得因为商务标页面状态区不同而省略告警。

下载必须照常发生：告警表示图片降级，不表示 Word 生成失败。前端不得增加二次确认、自动重试、重新上传、自动删除引用或阻止弹出下载。

## 4. 权限、网络与非目标

P9D 不新增路由、角色、鉴权、CSRF、API、任务类型、表、迁移、依赖或设置项；不修改后端导出、图片解析、Word 内容、文件下载、模板与项目图片协议。除既有导出任务和下载外，不新增网络请求，不访问图片外链，不写 `localStorage`、`sessionStorage`、IndexedDB、URL、剪贴板或日志。

本包不提供图片修复、失效引用定位到编辑器、批量替换、图片预览、商务标图片编辑入口、导出历史、下载进度或 WPS 自动打开；不扩大 `biaoshu-image://` 支持范围，也不允许 `http(s)`、`file:`、绝对路径或 data URL。

## 5. 验收底线

浏览器 E2E 至少覆盖：

1. 技术标含无效项目图片引用时，真实本机 export 任务成功，页面显示后端原因且仍触发本机下载地址；
2. 商务标 Markdown 含无效项目图片引用时行为一致；
3. 无告警的后续成功导出会清空旧告警；
4. 非数组、非字符串、空字符串、超量和超长输入被安全收敛，文本不得解释为 HTML 或链接；
5. 测试不得访问外网，不使用固定 sleep，Playwright 仍单 worker 串行。

回归必须通过前端 lint/build、P9D 定向 E2E 和全量单 worker E2E。后端无代码差异，已有项目图片定向测试作为后端权威契约回归。
