# 架构说明（前端阶段）

## 目标形态

```text
浏览器 (React)
    ↓ HTTP + SSE/WebSocket（后续）
FastAPI
    ↓
任务 Worker（长耗时生成/解析）
    ↓
核心服务：解析 / RAG / 流水线 / 导出
    ↓
PostgreSQL 或 SQLite · 文件存储 · Redis
```

当前仓库仅落地 **前端可交互原型**，业务数据为 mock。

## 技术方案工作流（对齐 C 端）

| 步骤 | 页面 | 产出 |
|------|------|------|
| 1 | 文档导入与解析 | 招标文件 Markdown |
| 2 | 招标分析 | 项目概述、技术要求、评分点 |
| 3 | 大纲编辑 | 三级目录 |
| 4 | 全局事实 | 抗幻觉事实清单 |
| 5 | 正文生成 | 分章 Markdown |
| 6 | 扩写与导出 | 字数达标 + Word |

## 本地解析插件

- 在线：轻量解析（服务端，后续实现）
- 本地：MinerU 助手安装 → Token 绑定 → 解析结果回传
- 结果格式见后续 `docs/parser-schema.md`（后端阶段补充）

## 工作空间

- `workspace` ≈ 当前登录账号（个人版）
- 所有项目、知识库、任务均挂在 workspace 下，为多成员扩展预留字段
